## Requisitos do Sistema
| Categoria | ID | Descrição do Requisito |
|--- | --- | ---|
| Funcional |RF - 01| O sistema deve permitir o gerenciamento (CRUD) de materiais e suas categorias. |
| Funcional | RF - 02 | O sistema deve registrar entradas e saídas (movimentações) do estoque. |
| Funcional | RF - 03 | O sistema deve permitir o gerenciamento completo de ativos patrimoniais.|
| Funcional | RF - 04 | O sistema deve restringir o acesso apenas a funcionários com NIF cadastrado no banco de dados.|
| Funcional | RF - 05 | O sistema deve validar o cadastro de novos usuários através de uma "lista branca" (pré-cadastro).| 
| Funcional | RF - 06 | O sistema deve permitir a visualização de inventários atualizados.| 
| Não Funcional | RNF - 01 | A interface de usuário deve seguir rigorosamente o Manual de Identidade Visual do SENAI. |
| Não Funcional| RNF - 02 | A API deve ser desenvolvida utilizando o framework Spring Boot com arquitetura RESTful. | 
| Não Funcional | RNF - 03 | O banco de dados deve ser normalizado e representar adequadamente as entidades e relacionamentos. | 
| Não Funcional | RNF - 04 | A interface deve ser intuitiva, amigável e responsiva. | 
| Não Funcional | RNF - 05 | O sistema deve utilizar persistência de dados através do Spring Data JPA.|
