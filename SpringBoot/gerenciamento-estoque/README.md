## Requisitos do Sistema
| Categoria | ID | Descrição do Requisito |
|--- | --- | ---|
| Funcional |RF01| O sistema deve permitir o gerenciamento (CRUD) de materiais e suas categorias. |
| Funcional | RF02 | O sistema deve registrar entradas e saídas (movimentações) do estoque. |
| Funcional | RF03 | O sistema deve permitir o gerenciamento completo de ativos patrimoniais.|
| Funcional | RF04 | O sistema deve restringir o acesso apenas a funcionários com NIF cadastrado no banco de dados.|
| Funcional | RF05 | O sistema deve validar o cadastro de novos usuários através de uma "lista branca" (pré-cadastro).| 
| Funcional | RF06 | O sistema deve permitir a visualização de inventários atualizados.| 
| Não Funcional | RNF01 | A interface de usuário deve seguir rigorosamente o Manual de Identidade Visual do SENAI. |
| Não Funcional| RNF02 | A API deve ser desenvolvida utilizando o framework Spring Boot com arquitetura RESTful. | 
| Não Funcional | RNF03 | O banco de dados deve ser normalizado e representar adequadamente as entidades e relacionamentos. | 
| Não Funcional | RNF04 | A interface deve ser intuitiva, amigável e responsiva. | 
| Não Funcional | RNF05 | O sistema deve utilizar persistência de dados através do Spring Data JPA.|
